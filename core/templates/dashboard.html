How to read this report

How scoring works

Scores summarize technical evidence. They are not verdicts, but fast indicators to guide attention.

  • Security Score combines DNSSEC data serving, cookies, EDNS, resolver restriction, web exposure, AXFR behavior, and CAA presence.
  • Privacy Score combines DoT, DoH, heuristic QNAME minimization, and ECS behavior.
  • Global Grade is derived from the available scores. When privacy does not apply, the grade becomes security-only.
  • N/A means the metric was not applicable or Phase 1 data was not available.

How to read the sections

  • Infrastructure shows server reachability, DNS service health, encryption support, resolver exposure, and capability markers. Analytics bar shows alive/dead/exposed counts at a glance.
  • Zones shows SOA visibility, synchronization, transfer exposure, DNSSEC, CAA, and check scope. Desync count is highlighted in the analytics bar.
  • Records shows consistency across repeated queries, transport context, answer previews, and semantic findings. Rows with findings show a yellow badge in the Sync column.
  • Cross-Phase Correlations groups flags by Infrastructure/Zones/Records per server, making multi-phase patterns immediately visible.
  • Coverage Reliability shows per-phase metric coverage with color-coded values: green for 90%+, amber for 70-89%, red for below 70%.

Interpretation hints

  • Open Resolver / PUBLIC is usually critical because the server answered a third-party recursive request from the public internet.
  • Zone Desync deserves attention when repeated across the same domain and multiple servers.
  • Divergence may be expected with load balancing, but repeated divergence can also reveal instability or inconsistent policy.
  • SOA_ONLY means the zone audit was incomplete, so findings must be interpreted carefully.

Manual verification

Most report fields can be validated manually. Click the ? icon on any column header or metric to see the explanation, then click "Verify Manually" to expand the command reference.

  • OS tabs switch between Linux and Windows commands for each verification step.
  • Click to copy any command block by clicking on it. The command is copied to your clipboard.
  • Tools used include dig, nslookup, ping, mtr, curl, openssl, kdig, nmap, and native PowerShell cmdlets.
  • Placeholders like <server_ip> and <domain> must be replaced with values from your report.
